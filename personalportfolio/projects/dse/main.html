Portal for viewing instrumentation of customer equipment

Company: Deep Sea Electronics

URL: http://www.livegenset.com/

This project is complete and has been fully handed over to the customer.

Note: Due to the sensitivity of the company's data, I have no credentials to provide for login.

Customer requirements

  • User account creation and secure login
  • Site navigation via an expanding hierarchical tree and customizable menu buttons.
  • Use of dials and other widgets to display real-time data from the equipment. It must be possible for the user to drag these widgets into position and save their changes.
  • The ability to capture and display events relating to a piece of equipment (eg: Oil Pressure Low). The user should be able to configure custom event triggers to capture only the events they are interested in.
  • Events that require user action (warnings and alarms), must be captured and displayed real-time in the form of on-screen popup boxes.
  • There must be a facility to automatic generate a PDF report, detailing the state of a piece of equipment. These reports need the option of being periodically e-mailed to the user.
  • There must be full language support allowing the user to submit a translation file for their chosen language resulting in all text in the portal being translated.

Instrumentation

  • Data from the industrial equipment was communicated to the server in a defined JSON structure.
  • On the receiving the data it was parsed and stored in a database.
  • The web portal could then periodically request data from the server to display on dials and other widgets.

Portal Configuration

  • The web portal has the ability to be customized in many ways.
  • This included the personalizing the colors and images used in the web portal, the language, the content of automatically generated reports and the filtering of notifications to name a few.
  • The user can choose which pieces of equipment they want to add to the portal for monitoring as well as choosing which aspects of the equipment they want to monitor via the removal and addition of data widgets.

Data protocols

Each piece of industrial equipment is fitted with a box of electronics which allows data to be received and transmitted to the web portal. Strict data protocols were devised to allow this data to be understood at both ends.

Data was transferred in the JSON format; For example:

{"ID":{"a":{"Alarm-ID":{"Timestamp":Alarm-Severity}}}}

Where parameters are clearly defined; For example:

Alarm-SeverityDescription
0No Severity
1Information
2Warning
3Shutdown
4Electrical Trip
5Engine
